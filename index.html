1. Types of Operating Systems

Explore the three specific types of operating systems required for 12.3.1. Click the tabs below to view detailed definitions and characteristics.

Real-Time Operating System (RTOS)

  • Definition: An OS intended to serve real-time applications that process data as it comes in, typically without buffer delays.
  • Key Feature: Must guarantee a response within a strict, specified time constraint (latency).
  • Types:
    • Hard Real-Time: Missing a deadline is a total system failure (e.g., car airbag, pacemaker).
    • Soft Real-Time: Missing a deadline degrades performance but isn't catastrophic (e.g., video streaming).
  • Usage: Industrial control systems, medical devices, aircraft autopilots.

Key Keywords

Guaranteed Response Safety Critical Redundancy Sensors/Actuators

Network Operating System (NOS)

  • Definition: An OS designed to support workstations, personal computers, and older terminals that are connected on a local area network (LAN).
  • Key Function: Manages network resources (files, printers) and security (user authentication).
  • Features:
    • Allows shared access to files, printers, and applications.
    • Centralized management of user accounts and security policies.
    • Provides network communication protocols.

Key Keywords

Resource Sharing Authentication File Server Concurrency

Batch Processing OS

  • Definition: A system where programs (jobs) are collected together in groups (batches) and processed automatically without user intervention.
  • Process: JCL (Job Control Language) is often used to instruct the OS.
  • Usage: Payroll processing, bank statement generation, billing systems.
  • Advantages: Can run during off-peak hours (night) to save computing resources; requires no operator presence.

Key Keywords

No User Interaction Off-peak JCL Sequential

Address Bus: Unidirectional. Carries the memory location address from CPU to Memory. Width determines max addressable memory (2^width).
Data Bus: Bidirectional. Carries the actual data/instructions between CPU and Memory. Width determines word size (bits per transfer).
Control Bus: Bidirectional/Unidirectional components. Carries control signals (Read, Write, Interrupt, Clock) to coordinate activities.
Performance Factors:
  • Clock Speed: Hz. Cycles per second. Higher = faster (usually).
  • Bus Width: Bits. Wider data bus = more data per fetch.
  • Word Size: Number of bits CPU processes in one go.

5. Memory Management

Paging vs Segmentation

Paging

Memory is divided into fixed-size physical blocks called 'Frames'. Logical memory is divided into same-sized 'Pages'.

  • Physical division.
  • User/Program doesn't see it.
  • Eliminates external fragmentation.

Segmentation

Memory is divided into variable-sized logical sections (e.g., Code, Stack, Data) based on the program structure.

  • Logical division.
  • Visible to the user/programmer.
  • Can cause external fragmentation.

6. Boolean Logic

Logic Laws

De Morgan's:

NOT(A AND B) = NOT A OR NOT B

NOT(A OR B) = NOT A AND NOT B

Distribution:

A AND (B OR C) = (A AND B) OR (A AND C)

Absorption:

A OR (A AND B) = A

A AND (A OR B) = A
